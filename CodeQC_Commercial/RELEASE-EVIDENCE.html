CodeQC Commercial Release Evidence

This page explains how to verify that the repository contains a binary-only commercial release and that the release has passed spec-mapped acceptance.

Evidence Chain

  1. Binary boundary: BINARY-ONLY-POLICY.html defines what may and may not be committed.
  2. Checksum: SHA256SUMS.txt pins the release ZIP hash.
  3. Manifest: release-manifest.json records package metadata, fixes, and validation scope.
  4. Acceptance contract: ACCEPTANCE-MATRIX.html and release-acceptance.json define CQC-COM-001 through CQC-COM-019.
  5. Product-owned test: codeqc-runner.exe selftest emits a JSON acceptance record with spec coverage.
  6. CI proof: GitHub Actions uploads codeqc-commercial-selftest for every commercial release gate run.

Acceptance Split

  • CQC-COM-001 and CQC-COM-002 are verified by the CI boundary gate.
  • CQC-COM-003 through CQC-COM-019 are verified by the commercial binary's selftest.
  • The CI gate fails if any acceptance ID in release-acceptance.json is missing from the combined evidence.

Manual Verification

Get-FileHash -Algorithm SHA256 .\releases\CodeQC_Commercial_Windows_x64_0.4.0.zip
Expand-Archive .\releases\CodeQC_Commercial_Windows_x64_0.4.0.zip -DestinationPath .\verify-codeqc
.\verify-codeqc\codeqc-runner.exe selftest --output .\selftest.json --keep-work-dir
Get-Content .\selftest.json

Non-Claims

This evidence does not claim code signing, installer readiness, license-server enforcement, long-duration load testing, or multi-OS release coverage.
